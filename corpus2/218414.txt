How to update web application deployed on intranet? There is a Java Spring application that will be deployed on intranet of multiple customers. In case when there is a major update what should the developer do in order to ensure that all the customers have the latest version of his software? There is an option that an update module is implemented in the application and it will automatically query the central web repository for updates. This option doesn't work if there is the server where the application is installed doesn't have internet connection. Another possibility that comes to mind is to manually send a .war file to the IT personnel of all the customers for them to install the latest version. How to deal with updates in this situation?
