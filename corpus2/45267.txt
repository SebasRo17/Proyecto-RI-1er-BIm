Why do `Permute` and `PermutationReplace` not invert one another, as stated in the documentation? If we have an identity permutation list (a list of the form `Range[n]`)               In[1]:= list1 = {1, 2, 3};      then `Permute` and `PermutationReplace` invert one another:               In[2]:= cycle = Cycles[{{1, 2, 3}}];             Permute[PermutationReplace[list1, cycle], cycle] == list1     Out[3]= True          In[4]:= PermutationReplace[Permute[list1, cycle], cycle] == list1     Out[4]= True      But this does not necessarily hold for lists not of the form `Range[n]`:               In[5]:= list2 = {3, 2, 1};             PermutationReplace[Permute[list2, cycle], cycle] == list2     Out[6]= False          In[7]:= Permute[PermutationReplace[list2, cycle], cycle] == list2     Out[7]= False      Why is this so?
